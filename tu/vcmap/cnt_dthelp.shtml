1. Introduction

VCMap is a tool that determines syntenic relationships between Rat, Mouse and Human and allows searching and visualization of these chromosomal regions in a variety of ways.

New to VCMap?-- Try the starter's guide, which which contains definitions and background information.

2. Starter's Guide


2.1 Definitions

Comparative Maping: Comparative mapping is a strategy for mapping sequence-related information from one species to another. It is based on the fact that the regions of sequence have been conserved among species during evolution (syntenic regions).

VCMap build: The core homology and mapping data that has been used to construct VCMap database. These include publicly available sequence data of genes, ESTs, their UniGene relationships, RH map marker information etc. from human, mouse and rat.

Static maps: Static maps are fixed image maps that are pregenerated from the VCMap database. They provide an overview of the entire chromosome but are not interactive. The static maps can be reached using the Static Maps link available on the top left hand side of most VCMap pages.

Dynamic maps: Dynamic maps are displayed using an interactive Java display. These can be fully customized by the user from Mapview page.

Framework markers: In genetic and radiation hybrid maps, these are markers used as reference markers for interspecies comparisons.

Backbone species: The species against which the other two species are aligned (For example, selecting rat as the backbone species, Chr1 as the chromosome and human/mouse as the non-backbone species would show the entire rat Chr1 with syntenic regions from the other species aligned to the rat.)

Bin predicition: A 'bin' prediction is made by predicting the location of the homologous UniGenes of the backbone species, based upon the mapped borders (breakpoints) of the homologous region of the reference (non-backbone) species.

Stream prediction: A 'stream' prediction, either upstream or downstream of the gene, is based upon the position of the homologous UniGene in the non-backbone species.

2.2 What can you do with VCMap?

Using VCMap, the user can generate:

  1. Visual representation of synteny between human, mouse and rats.
  2. Compare different maps available for the same species and compare them individually with types of maps for other two species.
  3. Obtain homologous marker information across these three species.
  4. Obtain a visual display of mapped markers (SSLPs, UniGenes and genes) in terms of both RHMaps and genetic maps of the three species; also, tabulated list of makers and related data can be obtained for any marker listed in the maps.
  5. Obtain detailed reports of individual markers on any chromosome from organism, including location, sequence, ESTs, genes, aliases, LOD scores of SSLP markers, corresponding homologous markers in the other two species compared to and corresponding links to external databases like GenBank, RGD etc.
  6. RHMap locations of the markers and genetic map location whenever available.

2.3 What are the limitations of VCMap?

  1. VCMap is constructed based on homologous UniGenes and other markers with known mapping information. So the map information is restricted to markers, which are sequenced and stored in various databases like RGD, GenBank, and UniGene etc. The physical map (from the genomic sequence) has not yet been integrated into the tool; however, work is in progress to do this.
  2. VCMap does not currently use the genomic sequence for generating virtual syntenic maps. Consequently the user cannot directly obtain information about the genomic sequence between the markers. Ensembl, NCBI and other databases provide sequence information of specific genomic regions.

By choosing the type of map, organism and chromosome, the user can view static comparative maps or dynamic maps. The User can define the search parameters to find the required data from the VCMap .
